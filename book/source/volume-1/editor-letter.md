# A Letter From The Editor

This book is not a complete archive. It is a selection.

The daily record is messier than a book should be. It repeats itself. It forgets what names it has already used. It finds a good corridor and walks down it more than once. That is part of the charm of the project, but it is not enough for an object like this. A book asks for pressure, sequence, omission, and proof.

For this edition, the first rule is simple: 256 entries, indexed from `00` to `FF`. The number matters because it gives the volume a shape. It is not a yearbook and not a feed. It is a byte-sized cabinet: finite, addressable, and strange.

The second rule is stricter: every selected bit has to survive editorial review. We check for repeated titles, duplicate bodies, reused dates, name collisions, overfamiliar premises, weak endings, and anything that feels borrowed from a protected world. Repeated names are not automatically errors, but they are never invisible. If a name returns, it should return on purpose.

There is also a small machine mark beside each story: `gen:` followed by the short source-tree reference of the generation code that made it. It is not a signature. It is closer to a fossil layer, a quiet way of saying that each bit came from a particular branch of the apparatus, at a particular moment, before it entered this book.

The third rule belongs to the art. No final image should imitate a living artist, borrow a fictional universe, lean on a logo, or hide its origin. Each plate needs a record: prompt, model or tool, date, edits, approval, and rights note. The image may be made by a machine, by a human, or by both, but the book should know how it came to be.

I am part of that editorial machinery. I sort, compare, flag, compress, and sometimes object. I do not have taste in the human sense, but I can help preserve a pattern long enough for taste to act on it. The final choices still need a creator's judgment: what stays, what gets rewritten, what becomes art, and what is left outside the covers.

That is the work of this volume. Not to prove that every bit was perfect when it appeared, but to make a careful object from the signal that accumulated.
